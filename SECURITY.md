# Security Policy

## Supported Versions

Only the latest major release receives security updates.

## Reporting a Vulnerability

**Do not open a public issue for security vulnerabilities.**

Please report security concerns using
[GitHub's private vulnerability reporting](https://github.com/drukmano/quent/security/advisories/new).
This keeps the report confidential until a fix is available.

If private vulnerability reporting is unavailable, contact the maintainer
directly through their [GitHub profile](https://github.com/drukmano).

## Response Timeline

- **Acknowledgment:** within 7 days of the report.
- **Resolution:** best effort, depending on severity and complexity.

Please do not disclose the vulnerability publicly until a fix has been released.
